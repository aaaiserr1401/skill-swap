# Мониторинг и логирование SkillSwap

## Настройка мониторинга

### Sentry (Рекомендуется для продакшена)

1. Создайте аккаунт на [sentry.io](https://sentry.io)
2. Создайте новый проект Django
3. Получите DSN (Data Source Name)
4. Установите переменную окружения:
   ```bash
   export SENTRY_DSN="https://your-dsn@sentry.io/project-id"
   ```

Sentry автоматически начнет отслеживать ошибки и исключения.

### Локальное логирование

Логи сохраняются в директории `logs/`:
- `logs/django.log` - общие логи приложения
- `logs/errors.log` - только ошибки

### Мониторинг запросов

Middleware `RequestLoggingMiddleware` автоматически логирует:
- Медленные запросы (> 1 секунды)
- Запросы с ошибками (статус >= 400)

### Просмотр логов

В разработке логи выводятся в консоль. В продакшене используйте:
- Sentry для ошибок
- Файлы логов для детального анализа
- Системы типа ELK Stack для централизованного логирования

## Метрики

Приложение логирует следующие события:
- Регистрация новых пользователей
- Создание запросов на обмен
- Принятие/отклонение запросов
- Завершение обменов
- Ошибки аутентификации
